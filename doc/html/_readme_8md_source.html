28 ## Format of Input
29 
30 1. To Get 3D object provide a *.txt* file containing the informaion about orthographic projections.
31  #### Format of *.txt* file ####
32  f 0 0 # a vertex (0, 0) in front projection
33  f 0 1 # a vertex (0, 1) in front projection
34  f ...
35  ...
36 
37  fe 1 2 # an edge in front projection between vertices (0, 0) and (0, 1)
38  fe ...
39  ...
40 
41  t 0 1 # a vertex (0, 1) in top projection
42  t 0 2 # a vertex (0, 2) in top projection
43  t 1 2 # a vertex (1, 2) in top projection
44  ...
45 
46  te 1 3 # an edge in side projection between vertices (0, 1) and (1, 2)
47  te 2 3 # an edge in side projection between vertices (0, 2) and (1, 2)
48  ...
49 
50  s 1 0 # a vertex (1, 0) in side projection
51  s 1 1 # a vertex (1, 1) in side projection
52  s ...
53  ...
54 
55  se 1 2 # an edge in side projection between vertices (1, 0) and (1, 1)
56  se ...
57  ...
58 
59  The application takes such a *.txt* file and generates a *.obj* file.
60 
61 2. To Generate Projections and Orthographic Views provide a *.obj* file of the 3D object.
62 
63  #### Format of *.obj* file ####
64  v 1 2 3 # a vertex 'v' followed by (x, y, z) coordinates of the point
65  v ...
66  ...
67 
68  f 1 3 4 # a face 'f' followed by indices of vertices lying on that plane
69  f ...
70  ...
71 
72  The application takes such a *.obj* file and generates a *.txt* file.
